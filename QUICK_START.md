# PetFit 快速启动指南

## 🚀 快速开始

### 1. 安装 Node.js
确保已安装 Node.js (v16 或更高版本)

### 2. 安装依赖
```bash
cd petfit
npm install
```

### 3. 启动开发服务器
```bash
npm run dev
```

浏览器访问: http://localhost:3000

## 📱 功能演示

### 首页 - 宠物互动
- 点击"喂食"增加宠物饥饿值
- 点击"玩耍"增加心情值
- 点击"清洁"提升宠物满意度
- 查看宠物等级和经验进度

### 数据页 - 运动统计
- 查看今日运动数据
- 切换周/月趋势图表
- 浏览成就徽章收集进度
- 回顾运动里程碑

### 运动页 - 设备连接
- 扫描蓝牙设备
- 连接/断开设备
- 快速开始运动
- 管理已连接设备

### 社群页 - 队伍排行
- 查看队伍排名
- 加入或创建队伍
- 邀请好友加入
- 查看活跃成员

### 个人页 - 设置管理
- 编辑个人资料
- 管理通知设置
- 查看数据统计
- 退出登录

## 🎨 设计特点

- **移动端优先**: 专为移动设备优化
- **渐变配色**: 粉色系渐变主题
- **流畅动画**: 页面切换和交互动画
- **响应式布局**: 适配不同屏幕尺寸

## 🛠️ 技术亮点

- **Vue 3 Composition API**: 更好的代码组织
- **TypeScript**: 类型安全
- **Pinia**: 轻量级状态管理
- **Tailwind CSS**: 原子化样式
- **SCSS**: 模块化样式

## 📂 项目结构说明

```
src/
├── views/          # 5个核心页面组件
├── stores/         # Pinia状态管理(宠物、用户)
├── router/         # 路由配置
├── api/            # HTTP请求封装
├── types/          # TypeScript类型定义
├── components/     # 可复用组件
└── styles/         # 全局样式
```

## 🔧 开发建议

1. **状态管理**: 使用 Pinia stores 管理全局状态
2. **组件复用**: 将通用UI抽取为组件
3. **类型安全**: 充分利用 TypeScript 类型系统
4. **样式规范**: Tailwind + SCSS 混合使用

## 📦 构建部署

```bash
# 构建生产版本
npm run build

# 预览生产版本
npm run preview
```

构建后的文件在 `dist/` 目录

## 🐛 常见问题

### Q: 端口被占用?
A: 修改 `vite.config.ts` 中的 port 配置

### Q: 依赖安装失败?
A: 尝试删除 `node_modules` 和 `package-lock.json`，重新安装

### Q: TypeScript 报错?
A: 确保 `tsconfig.json` 配置正确，运行 `npm run build` 检查
